# PulseFiles - Secure File Sharing Service

## Overzicht

**PulseFiles** is een veilige, tijdelijke file sharing service die volledig geïntegreerd is met PulseGuard. Het systeem stelt gebruikers in staat om bestanden veilig te delen via tijdelijke links, met automatische verwijdering na een ingestelde periode.

## Belangrijkste Features

### Veilig File Sharing
- **End-to-end Encryption**: Bestanden worden encrypted opgeslagen
- **Time-limited Access**: Automatische verwijdering na expiration
- **Secure Links**: Unieke, moeilijk te raden URLs
- **Access Logging**: Gedetailleerde logs van alle toegangspogingen

### PulseGuard Integratie
- **Rapport Sharing**: Deel monitoring rapporten en exports
- **Screenshot Storage**: Bewaar incident screenshots veilig
- **Log File Sharing**: Deel systeem logs met support teams
- **Backup Storage**: Tijdelijke opslag voor backups

### Analytics & Monitoring
- **Download Tracking**: Monitor hoeveel keer bestanden gedownload worden
- **Geographic Analytics**: Zie waar downloads vandaan komen
- **Security Monitoring**: Detecteer verdachte toegangspogingen
- **Usage Reports**: Gedetailleerde rapporten over storage usage

## API Specificaties

### File Upload
```bash
# Enkel bestand upload
POST /api/pulsefiles/upload
Content-Type: multipart/form-data

# Form data
file: <binary_file_data>
expiration_hours: 24
max_downloads: 5
password: optional_password
notify_email: user@example.com

# Response
{
  "file_id": "file-123456",
  "download_url": "https://files.pulsefiles.com/dl/abc123def456",
  "expires_at": "2024-01-18T10:30:00Z",
  "max_downloads": 5,
  "password_protected": false
}
```

### Batch Upload
```bash
# Multiple bestanden upload
POST /api/pulsefiles/upload/batch
Content-Type: multipart/form-data

# Form data
files: [<file1>, <file2>, <file3>]
expiration_hours: 48
compress_files: true
create_zip: true

# Response
{
  "batch_id": "batch-789012",
  "files": [
    {
      "file_id": "file-123456",
      "original_name": "report.pdf",
      "download_url": "https://files.pulsefiles.com/dl/abc123"
    }
  ],
  "zip_download_url": "https://files.pulsefiles.com/dl/zip-def456"
}
```

### File Management
```bash
# Haal file details op
GET /api/pulsefiles/{file_id}

# Update file settings
PUT /api/pulsefiles/{file_id}
Content-Type: application/json
{
  "expiration_hours": 72,
  "max_downloads": 10,
  "password": "new_password"
}

# Verwijder file
DELETE /api/pulsefiles/{file_id}

# Haal download statistics op
GET /api/pulsefiles/{file_id}/stats
```

### Download Process
```bash
# Password protected download
GET /api/pulsefiles/dl/{short_code}?password=mypassword

# Direct download (als geen password)
GET /api/pulsefiles/dl/{short_code}

# Response headers voor direct download
Content-Type: application/pdf
Content-Disposition: attachment; filename="report.pdf"
X-Download-Count: 3
X-Max-Downloads: 5
X-Expires-At: 2024-01-18T10:30:00Z
```

## Integratie Voorbeelden

### PulseGuard Report Sharing
```javascript
// Automatische upload van monitoring rapport
const shareMonitoringReport = async (reportData) => {
  const formData = new FormData();
  formData.append('file', reportData.blob, 'uptime-report.pdf');
  formData.append('expiration_hours', '168'); // 7 dagen
  formData.append('max_downloads', '50');
  formData.append('notify_email', 'team@company.com');

  const response = await fetch('/api/pulsefiles/upload', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${PULSEGUARD_API_KEY}`
    },
    body: formData
  });

  const result = await response.json();

  // Update PulseGuard report met share link
  await fetch(`/api/reports/${reportData.id}`, {
    method: 'PUT',
    headers: {
      'Authorization': `Bearer ${PULSEGUARD_API_KEY}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      share_url: result.download_url
    })
  });

  return result;
};
```

### Incident Screenshot Upload
```javascript
// Upload screenshot bij incident creatie
const uploadIncidentScreenshot = async (screenshotBlob, incidentId) => {
  const formData = new FormData();
  formData.append('file', screenshotBlob, 'incident-screenshot.png');
  formData.append('expiration_hours', '720'); // 30 dagen
  formData.append('max_downloads', '10');
  formData.append('password', generateSecurePassword());

  const uploadResponse = await fetch('/api/pulsefiles/upload', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${PULSEGUARD_API_KEY}`
    },
    body: formData
  });

  const uploadResult = await uploadResponse.json();

  // Koppel screenshot aan incident
  await fetch(`/api/incidents/${incidentId}/attachments`, {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${PULSEGUARD_API_KEY}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      type: 'screenshot',
      url: uploadResult.download_url,
      password_protected: true
    })
  });

  return uploadResult;
};
```

### CI/CD Artifact Sharing
```yaml
# GitHub Actions workflow voor artifact sharing
name: Deploy and Share Logs
on: [deployment]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Upload deployment logs
        run: |
          # Create log archive
          tar -czf deployment-logs.tar.gz /var/log/deployment/

          # Upload to PulseFiles
          curl -X POST "https://api.pulsefiles.com/api/upload" \
            -H "Authorization: Bearer ${{ secrets.PULSEFILES_API_KEY }}" \
            -F "file=@deployment-logs.tar.gz" \
            -F "expiration_hours=24" \
            -F "password=${{ secrets.LOGS_PASSWORD }}" \
            -o upload-result.json

          # Extract share URL
          SHARE_URL=$(cat upload-result.json | jq -r '.download_url')

          # Post to Slack
          curl -X POST "${{ secrets.SLACK_WEBHOOK }}" \
            -H 'Content-type: application/json' \
            -d "{\"text\":\"Deployment logs: $SHARE_URL\"}"
```

## Troubleshooting

### Veelvoorkomende Problemen

#### Upload Fails
```bash
# Controleer file size limits
ls -lh filename.ext

# Verificeer API key
curl -I -H "Authorization: Bearer YOUR_API_KEY" https://api.pulsefiles.com/api/upload

# Check quota
curl -H "Authorization: Bearer YOUR_API_KEY" https://api.pulsefiles.com/api/quota
```

#### Download Link Bestaat Niet
```bash
# Controleer expiration
curl -H "Authorization: Bearer YOUR_API_KEY" https://api.pulsefiles.com/api/files/{file_id}

# Check download count
# Response header X-Download-Count vs X-Max-Downloads

# Verify password (indien van toepassing)
curl -H "Authorization: Bearer YOUR_API_KEY" "https://api.pulsefiles.com/api/dl/{code}?password=yourpassword"
```

#### Storage Quota Overschreden
```bash
# Check current usage
curl -H "Authorization: Bearer YOUR_API_KEY" https://api.pulsefiles.com/api/usage

# List files by size
curl -H "Authorization: Bearer YOUR_API_KEY" https://api.pulsefiles.com/api/files?sort=size_desc

# Delete old files
curl -X DELETE -H "Authorization: Bearer YOUR_API_KEY" https://api.pulsefiles.com/api/files/{file_id}
```

### Debug Tools
```bash
# Upload test file
echo "test content" > test.txt
```

## Best Practices

### Security Best Practices
1. **Gebruik Sterke Passwords**: Voor gevoelige bestanden
2. **Stel Korte Expiratie Tijden In**: Voor tijdelijke shares
3. **Monitor Download Activity**: Controleer verdachte toegang
4. **Regular Cleanup**: Verwijder ongebruikte bestanden

### Performance Optimalisatie
1. **Comprimeer Grote Bestanden**: Gebruik ZIP voor multiple files
2. **Gebruik Batch Uploads**: Voor multiple gerelateerde bestanden
3. **Plan Expiration Strategisch**: Balans tussen accessibility en security
4. **Monitor Storage Usage**: Voorkom quota overschrijdingen

### Compliance Considerations
1. **Data Classification**: Classificeer bestanden op gevoeligheid
2. **Retention Policies**: Stel passende retention periods in
3. **Access Logging**: Bewaar audit logs voor compliance
4. **GDPR Compliance**: Respect data protection regulations

### Integration Best Practices
1. **Error Handling**: Implementeer proper error handling
2. **Retry Logic**: Voeg retry logic toe voor failed uploads
3. **Progress Tracking**: Toon upload/download progress
4. **Fallback Options**: Zorg voor alternatieven bij API uitval

PulseFiles biedt een veilige, betrouwbare manier om bestanden tijdelijk te delen, met volledige integratie in het PulseGuard ecosysteem. Met end-to-end encryptie, gedetailleerde analytics, en flexibele configuratie opties, is PulseFiles de ideale oplossing voor moderne file sharing behoeften.
